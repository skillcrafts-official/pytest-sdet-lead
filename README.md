# pytest-sdet-lead
Учебный проект для роста специализации Python SDET

## Проектирование тестов
Любой контрактный тест должен уметь отвечать на вопросы:
1. **Корректность структуры ответа**
   Какие поля есть у успешного профиля? Какие типы данных? Есть ли обязательные поля (`id`, `username` / `email`, `profile` с вложенными полями)?
   _Даже если контракт не описан, мы создаём его прямо в тесте как эталон._
2. **Идентичность данных**
   Запросив профиль пользователя, мы должны сверить, что ответ содержит именно те значения, которые лежат в базе (или сгенерированы фабрикой). Например, `response.data['email'] == user.email`.
3. **Контракт ошибок**
   Для 401, 404, 403 тело ответа должно содержать стандартный DRF-объект с ключом `detail` (или кастомный). Проверяем не только статус, но и сообщение.
4. **Граничные условия**
   Что вернёт запрос `pk=0`, `pk=-1`, `pk=строка`? А если `pk` в UUID-формате (если модель использует UUID)? Мы можем добавить пару кейсов.
